RAG & AI Agent

Deux concepts fondamentaux de l'IA moderne, expliqués simplement avec du code concret.

RAG — Retrieval-Augmented Generation

Le RAG est une technique qui donne à un modèle IA la capacité de chercher dans des documents avant de répondre — au lieu de se baser uniquement sur ce qu'il a appris lors de son entraînement.

```
📚
Analogie simple

Imagine un étudiant qui passe un examen. Un LLM classique répond de mémoire. Un LLM avec RAG peut ouvrir ses notes et chercher la bonne page avant de répondre. Le résultat est bien plus précis et à jour.

Le flux de fonctionnement :

1
❓ Question
de l'utilisateur
→
2
🔍 Recherche
dans la base docs
→
3
📄 Extraits
pertinents récupérés
→
4
🤖 LLM génère
une réponse ciblée

💡 Cas d'usage : chatbot sur tes propres documents PDF, base de connaissances interne d'entreprise, assistant juridique/médical sur des textes spécifiques.

```

AI Agent — Agent Autonome

Un AI Agent est un système IA capable de prendre des décisions et d'agir de façon autonome pour accomplir un objectif. Il peut utiliser des outils, exécuter du code, naviguer sur le web, et enchaîner plusieurs étapes.

```
🤖
Analogie simple

Un LLM classique = un conseiller qui te donne des recommandations. Un AI Agent = un assistant personnel qui exécute réellement les tâches à ta place : il réserve le vol, envoie l'email, met à jour le tableau de bord.

La boucle de raisonnement (ReAct loop) :

1
🎯 Objectif
défini
→
2
🧠 Réflexion :
quelle action ?
→
3
🔧 Appel
d'un outil
→
4
📊 Observation
du résultat
→
5
🔁 Répète
jusqu'au but

💡 Cas d'usage : agent de recherche qui cherche sur le web et résume, agent DevOps qui déploie et monitore, agent commercial qui lit des emails et met à jour un CRM.

```

RAG vs AI Agent — Quelle différence ?

Critère 🟢 RAG 🟠 AI Agent
Rôle principal Chercher & répondre Planifier & agir
Autonomie Faible — réactif Élevée — proactif
Outils utilisés Moteur de recherche vectoriel Web, code, API, fichiers…
Complexité Modérée Élevée
Idéal pour Questions sur des documents Tâches multi-étapes
Peut contenir un RAG ? — ✅ Oui, souvent !

Comment les créer — Code Python

rag_simple.py — LangChain + ChromaDB Python 
# ─────────────────────────────────────────────
```

# ÉTAPE 1 : Installer les dépendances

# pip install langchain chromadb openai langchain-openai

# ─────────────────────────────────────────────

from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain.chains import RetrievalQA

# ─── 1. Charger ton document ───────────────────
loader = TextLoader(“mon_document.txt”)
documents = loader.load()

# ─── 2. Découper en morceaux (chunks) ──────────
splitter = RecursiveCharacterTextSplitter(
chunk_size=500,      # 500 caractères par morceau
chunk_overlap=50     # chevauchement pour ne rien perdre
)
chunks = splitter.split_documents(documents)

# ─── 3. Créer la base vectorielle ──────────────
# Les embeddings transforment le texte en vecteurs numériques
vectordb = Chroma.from_documents(
documents=chunks,
embedding=OpenAIEmbeddings()
)

# ─── 4. Créer la chaîne RAG ────────────────────
llm = ChatOpenAI(model=“gpt-4o-mini”)

rag_chain = RetrievalQA.from_chain_type(
llm=llm,
retriever=vectordb.as_retriever(search_kwargs={“k”: 3}),
)

# ─── 5. Poser une question ! ───────────────────
question = “Quelles sont les principales conclusions du document ?”
reponse = rag_chain.invoke(question)
print(reponse[“result”])
# → Le modèle cherche dans tes docs et répond précisément !

agent_simple.py — LangChain Agent avec outils Python 
# ─────────────────────────────────────────────
```

# ÉTAPE 1 : Installer les dépendances

# pip install langchain langchain-openai duckduckgo-search

# ─────────────────────────────────────────────

from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain.tools import Tool
from langchain_community.tools import DuckDuckGoSearchRun
from langchain import hub

# ─── 1. Définir les outils de l’agent ──────────
search_tool = DuckDuckGoSearchRun()

# Tu peux aussi créer des outils personnalisés !
def calculer(expression: str) -> str:
try:
return str(eval(expression))
except:
return “Erreur de calcul”

tools = [
Tool(
name=“Recherche Web”,
func=search_tool.run,
description=“Cherche des infos actuelles sur internet”
),
Tool(
name=“Calculatrice”,
func=calculer,
description=“Calcule une expression mathématique”
),
]

# ─── 2. Charger le prompt ReAct ────────────────
# ReAct = Reason + Act (raisonner puis agir en boucle)
prompt = hub.pull(“hwchase17/react”)

# ─── 3. Créer l’agent ──────────────────────────
llm = ChatOpenAI(model=“gpt-4o-mini”, temperature=0)

agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(
agent=agent,
tools=tools,
verbose=True,    # affiche le raisonnement étape par étape
max_iterations=5
)

# ─── 4. Lancer l’agent sur une tâche ───────────
tache = “Trouve le prix actuel du Bitcoin et calcule combien
vaut 0.5 BTC en dollars.”

resultat = executor.invoke({“input”: tache})
print(resultat[“output”])
# → L’agent cherche, calcule, et te donne la réponse finale !

Conseils pour bien démarrer

🧱

Commence par RAG

Plus simple à mettre en place. Idéal comme premier projet concret avec un LLM.

🔑

Clé API OpenAI

Les exemples utilisent OpenAI. Tu peux aussi utiliser Mistral, Claude (Anthropic), ou des modèles locaux (Ollama).

📦

LangChain = le couteau suisse

C'est la lib Python de référence pour connecter LLMs, outils, bases vectorielles et agents.

🗄️

Bases vectorielles

ChromaDB (local), Pinecone (cloud), Qdrant. Elles stockent les embeddings pour la recherche sémantique du RAG.

🚀

Frameworks Agents

LangGraph, CrewAI, AutoGen pour des agents plus avancés avec plusieurs rôles et collaborations.

⚠️

Attention aux coûts

Les appels API OpenAI sont payants. Commence avec gpt-4o-mini (moins cher) pour tester.
