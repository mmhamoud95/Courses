01 / INTRO

Introduction à LangGraph

Pourquoi LangGraph ? Qu'est-ce qu'un graphe d'état ?

```

Qu'est-ce que LangGraph ?

LangGraph est un framework de la suite LangChain qui modélise les workflows LLM comme des graphes orientés. Là où LangChain impose une exécution séquentielle, LangGraph permet des boucles, des conditions, du parallélisme et une mémoire persistante.

  • Chaque nœud est une fonction Python qui transforme l'état
  • Les arêtes définissent les transitions (fixes ou conditionnelles)
  • L'état (StateGraph) est un dictionnaire partagé entre tous les nœuds
  • Le checkpointer permet de sauvegarder et reprendre une exécution
◈

StateGraph

Le graphe principal. Déclare les nœuds, les arêtes et l'état global.

⬡

State

TypedDict partagé. Chaque nœud le lit et retourne ses modifications.

⚙

Nodes

Fonctions Python pures : (state) → dict. Elles ne retournent que les champs modifiés.

⟶

Edges

Transitions fixes (add_edge) ou conditionnelles (add_conditional_edges).

from langgraph.graph import StateGraph, START, END, add_messages
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

# ── 1. Définir l’état ─────────────────────────────────────────
class SimpleState(TypedDict):
messages: Annotated[list, add_messages]   # réducteur : accumule
compteur: int

# ── 2. Créer le LLM ───────────────────────────────────────────
llm = ChatOpenAI(model=“gpt-4o”, temperature=0)

# ── 3. Définir les nœuds ──────────────────────────────────────
def call_model(state: SimpleState) -> dict:
””“Appelle le LLM avec les messages actuels.”””
response = llm.invoke(state[“messages”])
return {
“messages”: [response],              # add_messages l’ajoute
“compteur”: state[“compteur”] + 1
}

# ── 4. Construire le graphe ────────────────────────────────────
builder = StateGraph(SimpleState)

builder.add_node(“model”, call_model)
builder.add_edge(START, “model”)
builder.add_edge(“model”, END)

app = builder.compile()

# ── 5. Exécuter ───────────────────────────────────────────────
result = app.invoke({
“messages”: [HumanMessage(content=“Bonjour, qui es-tu ?”)],
“compteur”: 0
})

print(result[“messages”][-1].content)
print(f”Appels LLM : {result[‘compteur’]}”)

02 / STATE

State & Réducteurs

Le cœur de LangGraph : comment définir et faire évoluer l'état.

```

Comment fonctionne l'état ?

L'état est un TypedDict qui voyage entre les nœuds. Chaque nœud reçoit l'état complet mais ne retourne que les champs modifiés. LangGraph fusionne ensuite les modifications dans l'état global grâce aux réducteurs.

  • Sans réducteur : la nouvelle valeur remplace l'ancienne
  • Avec réducteur : la nouvelle valeur est fusionnée selon la logique du réducteur
  • Annotated[list, operator.add] → concatène les listes
  • Annotated[list, add_messages] → spécial messages (gère IDs, suppressions)
python — state_avancé.py 
from typing import TypedDict, Annotated, Optional
```

from langgraph.graph import add_messages
import operator

class WorkflowState(TypedDict):
# ── Avec réducteur add_messages (recommandé pour les messages) ──
messages: Annotated[list, add_messages]

```
# ── Avec réducteur operator.add (concatène les listes) ──────────
logs: Annotated[list, operator.add]

# ── Sans réducteur : remplacement direct ────────────────────────
current_task: str
iteration:    int
is_done:      bool

# ── Champs optionnels ────────────────────────────────────────────
final_answer: Optional[str]

# ── Réducteur personnalisé : garde uniquement les 5 derniers ────
history: Annotated[list, lambda old, new: (old + new)[-5:]]
```

# ── Exemple : deux nœuds qui modifient l’état ────────────────────
def node_a(state: WorkflowState) -> dict:
# Retourne SEULEMENT les champs modifiés
return {
“current_task”: “processing”,
“logs”:         [“node_a exécuté”],   # sera concaténé
“iteration”:   state[“iteration”] + 1
}

def node_b(state: WorkflowState) -> dict:
return {
“final_answer”: “Réponse calculée”,
“is_done”:      True,
“logs”:         [“node_b exécuté”]    # sera ajouté à la liste
}
# État final des logs : [“node_a exécuté”, “node_b exécuté”]
```

03 / NODES

Nœuds & Arêtes

Construire le graphe, brancher les nœuds, créer des cycles.

```

Types d'arêtes

  • add_edge(A, B) — Transition fixe : A exécute toujours B ensuite
  • add_conditional_edges(A, fn) — La fonction fn(state) retourne le nom du prochain nœud
  • add_edge(A, END) — Termine le graphe depuis le nœud A
  • add_edge(START, A) — Point d'entrée du graphe
Graphe linéaire (pipeline)
python — pipeline.py 
from langgraph.graph import StateGraph, START, END
```

# Pipeline : START → preprocess → analyze → format → END

def preprocess(state): return {“text”: state[“text”].strip().lower()}
def analyze(state):    return {“score”: len(state[“text”].split())}
def format_output(state): return {“result”: f”Score: {state[‘score’]} mots”}

builder = StateGraph(PipelineState)

builder.add_node(“preprocess”,    preprocess)
builder.add_node(“analyze”,       analyze)
builder.add_node(“format_output”, format_output)

builder.add_edge(START,           “preprocess”)
builder.add_edge(“preprocess”,    “analyze”)
builder.add_edge(“analyze”,       “format_output”)
builder.add_edge(“format_output”, END)

app = builder.compile()

04 / ROUTING

Décisions & Routage

Construire des workflows intelligents avec branches conditionnelles.

```

add_conditional_edges — la puissance de LangGraph

La fonction de routage reçoit l'état et retourne une string correspondant à un nœud cible. Elle peut être un LLM, une logique Python, ou les deux.

  • Retourner END (ou "__end__") termine le graphe
  • Le mapping optionnel permet de renommer les clés retournées
  • La fonction peut retourner une liste pour des branches parallèles
Exemple complet : Analyseur de requêtes
python — analyseur_requetes.py 
import json
```

from typing import TypedDict, Annotated, Literal
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage
from langgraph.graph import StateGraph, START, END, add_messages

llm = ChatOpenAI(model=“gpt-4o”, temperature=0)

class QueryState(TypedDict):
messages:  Annotated[list, add_messages]
query:     str
category:  str    # “math” | “search” | “code” | “general”
response:  str

# ── Nœud de classification ────────────────────────────────────
def classify(state: QueryState) -> dict:
””“Utilise le LLM pour classifier la requête.”””
prompt = f””“Classifie cette requête. Réponds UNIQUEMENT avec ce JSON :
{{“category”: “math”|“search”|“code”|“general”}}

Requête : {state[‘query’]}”””

```
resp = llm.invoke([HumanMessage(content=prompt)])
data = json.loads(resp.content)
return {"category": data["category"]}
```

# ── Nœuds spécialisés ─────────────────────────────────────────
def handle_math(state: QueryState) -> dict:
resp = llm.invoke([
SystemMessage(content=“Expert mathématiques. Résous étape par étape.”),
HumanMessage(content=state[“query”])
])
return {“response”: f”[MATH] {resp.content}”}

def handle_search(state: QueryState) -> dict:
resp = llm.invoke([
SystemMessage(content=“Synthétise une réponse factuelle et sourcée.”),
HumanMessage(content=state[“query”])
])
return {“response”: f”[SEARCH] {resp.content}”}

def handle_code(state: QueryState) -> dict:
resp = llm.invoke([
SystemMessage(content=“Expert développeur. Fournis du code propre et commenté.”),
HumanMessage(content=state[“query”])
])
return {“response”: f”[CODE] {resp.content}”}

def handle_general(state: QueryState) -> dict:
resp = llm.invoke([HumanMessage(content=state[“query”])])
return {“response”: resp.content}

# ── Routeur ───────────────────────────────────────────────────
def route_by_category(state: QueryState) -> Literal[“math”,“search”,“code”,“general”]:
return state[“category”]   # retourne directement la catégorie

# ── Assemblage ────────────────────────────────────────────────
builder = StateGraph(QueryState)

builder.add_node(“classify”, classify)
builder.add_node(“math”,     handle_math)
builder.add_node(“search”,   handle_search)
builder.add_node(“code”,     handle_code)
builder.add_node(“general”,  handle_general)

builder.add_edge(START, “classify”)
builder.add_conditional_edges(“classify”, route_by_category)
# Chaque branche se termine
for node in [“math”, “search”, “code”, “general”]:
builder.add_edge(node, END)

app = builder.compile()

# ── Test ──────────────────────────────────────────────────────
result = app.invoke({
“messages”: [],
“query”:    “Quelle est la dérivée de x³ + 2x ?”,
“category”: ””,
“response”: ””
})
print(result[“response”])
# → [MATH] La dérivée de x³ + 2x est 3x² + 2

Agents avec Outils

Pattern ReAct moderne — sans create_react_agent déprécié.

```
⚠
create_react_agent est déprécié. N'utilisez plus from langgraph.prebuilt import create_react_agent. Le pattern manuel ci-dessous offre un contrôle total et est la méthode recommandée pour toute application sérieuse.

Pattern ReAct manuel

La boucle agent suit ce schéma :

  • 1. Le LLM analyse les messages et génère soit une réponse finale, soit des tool_calls
  • 2. Si tool_calls → exécuter les outils → ajouter les ToolMessage → retour au LLM
  • 3. Si réponse finale → END
python — agent_outils_complet.py 
from typing import TypedDict, Annotated, Literal
```

from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage, SystemMessage
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, START, END, add_messages
import math, json

# ── Définition des outils ─────────────────────────────────────
@tool
def calculatrice(expression: str) -> str:
””“Évalue une expression mathématique Python sûre.

```
Args:
    expression: Ex: "2 ** 10", "math.sqrt(144)", "sum([1,2,3])"
"""
try:
    result = eval(expression, {"__builtins__": {}, "math": math})
    return f"Résultat : {result}"
except Exception as e:
    return f"Erreur : {e}"
```

@tool
def meteo(ville: str) -> str:
””“Retourne la météo actuelle d’une ville.

```
Args:
    ville: Nom de la ville (ex: "Paris", "Lyon")
"""
# En production : appeler une vraie API météo
meteos = {"paris": "☀️ 22°C, ensoleillé", "lyon": "🌤 18°C, nuageux"}
return meteos.get(ville.lower(), f"Données indisponibles pour {ville}")
```

@tool
def recherche(query: str) -> str:
””“Effectue une recherche et retourne un résumé.

```
Args:
    query: Terme ou question à rechercher
"""
# En production : appeler Tavily, Serper, etc.
return f"Résultats de recherche pour '{query}': [données simulées]"
```

# ── Configuration ─────────────────────────────────────────────
tools           = [calculatrice, meteo, recherche]
tools_by_name   = {t.name: t for t in tools}
llm             = ChatOpenAI(model=“gpt-4o”, temperature=0)
llm_with_tools  = llm.bind_tools(tools)   # ← méthode moderne

# ── État ──────────────────────────────────────────────────────
class AgentState(TypedDict):
messages:  Annotated[list, add_messages]
iteration: int

# ── Nœud Agent ───────────────────────────────────────────────
def agent(state: AgentState) -> dict:
””“Appelle le LLM avec les outils disponibles.”””
response = llm_with_tools.invoke(state[“messages”])
return {
“messages”:  [response],
“iteration”: state[“iteration”] + 1
}

# ── Nœud Tools ───────────────────────────────────────────────
def execute_tools(state: AgentState) -> dict:
””“Exécute tous les outils demandés par le LLM.”””
last_message  = state[“messages”][-1]
tool_messages = []

```
for tool_call in last_message.tool_calls:
    tool_fn = tools_by_name[tool_call["name"]]
    result  = tool_fn.invoke(tool_call["args"])

    tool_messages.append(ToolMessage(
        content=str(result),
        tool_call_id=tool_call["id"],
        name=tool_call["name"]
    ))

return {"messages": tool_messages}
```

# ── Routeur ───────────────────────────────────────────────────
def should_continue(state: AgentState) -> Literal[“tools”, ”**end**”]:
if state[“iteration”] >= 10:          # sécurité
return END
last = state[“messages”][-1]
if hasattr(last, “tool_calls”) and last.tool_calls:
return “tools”
return END

# ── Assemblage ────────────────────────────────────────────────
builder = StateGraph(AgentState)

builder.add_node(“agent”, agent)
builder.add_node(“tools”, execute_tools)

builder.add_edge(START, “agent”)
builder.add_conditional_edges(“agent”, should_continue)
builder.add_edge(“tools”, “agent”)

app = builder.compile()

# ── Test multi-outils ─────────────────────────────────────────
result = app.invoke({
“messages”: [HumanMessage(
content=“Quelle est la météo à Paris ? Et combien font 2^32 ?”
)],
“iteration”: 0
})

for msg in result[“messages”]:
kind = type(msg).**name**
content = msg.content[:80] if msg.content else ”[tool_calls]”
print(f”{kind:15} │ {content}”)
06 / SUPERVISOR

Pattern Superviseur

Architecture multi-agents avec orchestration centralisée.

```

Architecture Superviseur

Un nœud superviseur (LLM) reçoit la tâche et décide quel agent spécialisé appeler. Après chaque agent, le contrôle revient au superviseur qui peut appeler un autre agent ou terminer.

  • Le superviseur utilise un structured output pour choisir le prochain agent
  • Chaque agent est un nœud indépendant avec ses propres outils
  • Le superviseur peut enchaîner plusieurs agents sur la même tâche
  • L'état commun contient les résultats de tous les agents
ARCHITECTURE SUPERVISEUR ┌──────────────────┐ START ────────► │ SUPERVISEUR │ ◄─────────────────────────┐ │ (LLM router) │ │ └─────────┬────────┘ │ │ │ ┌─────────────────────┼──────────────────┐ │ ▼ ▼ ▼ │ ┌─────────────────┐ ┌────────────────┐ ┌────────────────┐ │ │ RESEARCHER │ │ ANALYST │ │ WRITER │ │ │ (recherche web) │ │ (données/SQL) │ │ (rédaction) │ │ └────────┬────────┘ └───────┬────────┘ └───────┬────────┘ │ └─────────────────────┴──────────────────┘ │ │ résultats → state │ └──────────────────────────────────────────┘ │ FINISH ▼ END
python — superviseur_complet.py 
from typing import TypedDict, Annotated, Literal
```

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage, AIMessage
from langgraph.graph import StateGraph, START, END, add_messages
import json

llm = ChatOpenAI(model=“gpt-4o”, temperature=0)

AGENTS = [“researcher”, “analyst”, “writer”, “FINISH”]

class SupervisorState(TypedDict):
messages:       Annotated[list, add_messages]
task:           str
agent_outputs:  Annotated[dict, lambda a, b: {**a, **b}]  # merge dicts
next_agent:     str
final_response: str

# ── Nœud Superviseur ─────────────────────────────────────────
SUPERVISOR_PROMPT = ””“Tu es un superviseur orchestrant ces agents : {agents}.

Tâche en cours : {task}

Résultats disponibles : {outputs}

Décide quel agent appeler ENSUITE pour progresser vers la solution.
Réponds UNIQUEMENT avec ce JSON : {{“next”: “researcher”|“analyst”|“writer”|“FINISH”}}

Appelle FINISH quand la tâche est complète.”””

def supervisor(state: SupervisorState) -> dict:
””“Décide quel agent appeler.”””
prompt = SUPERVISOR_PROMPT.format(
agents  = ”, “.join(AGENTS),
task    = state[“task”],
outputs = json.dumps(state[“agent_outputs”], ensure_ascii=False, indent=2)
)
resp = llm.invoke([HumanMessage(content=prompt)])
data = json.loads(resp.content)
return {“next_agent”: data[“next”]}

# ── Routeur depuis le superviseur ────────────────────────────
def route_to_agent(state: SupervisorState):
return END if state[“next_agent”] == “FINISH” else state[“next_agent”]

# ── Agents spécialisés ────────────────────────────────────────
def researcher(state: SupervisorState) -> dict:
””“Agent de recherche d’informations.”””
resp = llm.invoke([
SystemMessage(content=“Expert en recherche. Collecte des faits pertinents.”),
HumanMessage(content=state[“task”])
])
return {“agent_outputs”: {“researcher”: resp.content}}

def analyst(state: SupervisorState) -> dict:
””“Agent d’analyse et de synthèse.”””
context = state[“agent_outputs”].get(“researcher”, ””)
resp = llm.invoke([
SystemMessage(content=“Expert en analyse. Structure et interprète les données.”),
HumanMessage(content=f”Contexte : {context}\n\nTâche : {state[‘task’]}”)
])
return {“agent_outputs”: {“analyst”: resp.content}}

def writer(state: SupervisorState) -> dict:
””“Agent rédacteur — produit la réponse finale.”””
all_context = json.dumps(state[“agent_outputs”], ensure_ascii=False)
resp = llm.invoke([
SystemMessage(content=“Expert en rédaction. Synthétise en réponse claire et structurée.”),
HumanMessage(content=f”Contexte : {all_context}\n\nTâche : {state[‘task’]}”)
])
return {
“agent_outputs”:  {“writer”: resp.content},
“final_response”: resp.content
}

# ── Assemblage ────────────────────────────────────────────────
builder = StateGraph(SupervisorState)

builder.add_node(“supervisor”, supervisor)
builder.add_node(“researcher”, researcher)
builder.add_node(“analyst”,    analyst)
builder.add_node(“writer”,     writer)

builder.add_edge(START, “supervisor”)
builder.add_conditional_edges(“supervisor”, route_to_agent)

# Après chaque agent → retour au superviseur
for agent_name in [“researcher”, “analyst”, “writer”]:
builder.add_edge(agent_name, “supervisor”)

app = builder.compile()

# ── Utilisation ───────────────────────────────────────────────
result = app.invoke({
“messages”:      [],
“task”:          “Analyse les tendances de l’IA en 2025 et rédige un rapport concis.”,
“agent_outputs”: {},
“next_agent”:    ””,
“final_response”: ””
})

print(”=== RAPPORT FINAL ===”)
print(result[“final_response”])
07 / HITL

Human-in-the-Loop

Interruptions, validation humaine et reprise d'exécution.

```

Pourquoi interrompre un graphe ?

LangGraph permet de mettre en pause l'exécution pour une intervention humaine (validation, correction, approbation) puis de reprendre là où le graphe s'est arrêté. Cela nécessite un checkpointer pour sauvegarder l'état.

  • interrupt_before=["node"] — pause AVANT l'exécution du nœud
  • interrupt_after=["node"] — pause APRÈS l'exécution du nœud
  • app.update_state() — modifie l'état avant de reprendre
  • app.invoke(None, config) — reprend l'exécution
python — human_in_the_loop.py 
from langgraph.checkpoint.memory import MemorySaver
```

from langgraph.graph import StateGraph, START, END, add_messages
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage
from typing import TypedDict, Annotated

llm       = ChatOpenAI(model=“gpt-4o”)
checkpointer = MemorySaver()

class ReviewState(TypedDict):
messages:      Annotated[list, add_messages]
draft:         str
approved:      bool
human_feedback: str

# ── Nœuds ────────────────────────────────────────────────────
def generate_draft(state: ReviewState) -> dict:
””“Génère un brouillon.”””
last_msg = state[“messages”][-1]
resp = llm.invoke([last_msg])
return {
“messages”: [resp],
“draft”:    resp.content
}

def revise_draft(state: ReviewState) -> dict:
””“Révise en tenant compte du feedback humain.”””
resp = llm.invoke([
HumanMessage(content=f””“Brouillon original :

11 / SUBGRAPHS

Subgraphs

Graphes imbriqués pour organiser des workflows complexes en modules réutilisables.

```

Pourquoi utiliser des Subgraphs ?

Un subgraph est un graphe LangGraph complet utilisé comme nœud dans un graphe parent. Cela permet de :

  • Décomposer un workflow complexe en modules indépendants et testables
  • Réutiliser un même sous-workflow dans plusieurs graphes parents
  • Isoler la logique d'un agent ou d'un pipeline
  • Partager partiellement l'état entre parent et enfant (les clés communes sont transmises)
ARCHITECTURE SUBGRAPH ``` GRAPHE PARENT ┌──────────────────────────────────────────────────────┐ │ │ │ START ──► [ intake ] ──► [ SUBGRAPH: research ] ──► [ writer ] ──► END │ │ │ │ │ │ SUBGRAPH (graphe interne) │ │ │ ┌──────────────────────┐ │ │ └─►│ START │ │ │ │ → search │ │ │ │ → validate │ │ │ │ → summarize → END │ │ │ └──────────────────────┘ │ └──────────────────────────────────────────────────────┘
```
Exemple : Subgraph de validation

Parallélisme

Exécuter plusieurs nœuds simultanément avec Send() et les branches parallèles.

```

Deux approches de parallélisme

  • Branches statiques : retourner une liste de nœuds depuis add_conditional_edges — exécution parallèle synchronisée sur un fan-in
  • Send() dynamique : créer des branches à la volée avec Send(node, state) — idéal pour mapper sur une liste d'items
FAN-OUT / FAN-IN (parallélisme statique) [ split ] / │ \ ▼ ▼ ▼ [task_a] [task_b] [task_c] ← exécution parallèle \ │ / ▼ ▼ ▼ [ merge ] ──► END ``` SEND() DYNAMIQUE (map sur une liste) [ planner ] ──► Send(“worker”, item) × N ← N branches créées dynamiquement │ ┌───────────┼───────────┐ ▼ ▼ ▼ [worker] [worker] [worker] └───────────┴───────────┘ ▼ [ aggregate ]

Structured Output & Pydantic

Extraire des données typées depuis le LLM avec with_structured_output.

```

Pourquoi le Structured Output ?

Plutôt que de parser du JSON fragile depuis le contenu texte, LangChain propose llm.with_structured_output() qui utilise le function calling pour garantir un output typé. En combinaison avec Pydantic, on obtient une validation automatique.

  • Aucun parsing JSON manuel → pas de crash sur un format inattendu
  • Validation Pydantic automatique (types, champs requis, valeurs par défaut)
  • Idéal pour les routeurs, classifieurs et extracteurs dans LangGraph
python — structured_output.py 
from pydantic import BaseModel, Field
```

from typing import TypedDict, Annotated, Literal, Optional
from langgraph.graph import StateGraph, START, END, add_messages
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

llm = ChatOpenAI(model=“gpt-4o”, temperature=0)

# ── Schémas Pydantic ──────────────────────────────────────────
class TaskClassification(BaseModel):
””“Classification d’une requête utilisateur.”””
category:   Literal[“math”, “code”, “search”, “general”]
confidence: float = Field(ge=0.0, le=1.0, description=“Score de confiance”)
reasoning:  str   = Field(description=“Justification de la classification”)
subtasks:   list[str] = Field(default_factory=list, description=“Sous-tâches détectées”)

class DocumentSummary(BaseModel):
””“Résumé structuré d’un document.”””
title:      str
key_points: list[str] = Field(min_length=1, max_length=5)
sentiment:  Literal[“positif”, “négatif”, “neutre”]
word_count: Optional[int] = None

class RouterDecision(BaseModel):
””“Décision du superviseur.”””
next_agent: Literal[“researcher”, “analyst”, “writer”, “FINISH”]
reason:     str

# ── LLMs avec structured output ───────────────────────────────
classifier_llm = llm.with_structured_output(TaskClassification)
summarizer_llm = llm.with_structured_output(DocumentSummary)
router_llm     = llm.with_structured_output(RouterDecision)

# ── Utilisation dans des nœuds LangGraph ──────────────────────
class StructuredState(TypedDict):
messages:       Annotated[list, add_messages]
query:          str
classification: Optional[dict]
next_agent:     str

def classify_node(state: StructuredState) -> dict:
””“Classifie avec output structuré garanti.”””
result: TaskClassification = classifier_llm.invoke([
HumanMessage(content=f”Classifie cette requête : {state[‘query’]}”)
])
# result est un objet Pydantic validé, pas du JSON brut
print(f”Catégorie : {result.category} (confiance : {result.confidence:.0%})”)
print(f”Raison : {result.reasoning}”)
return {“classification”: result.model_dump()}

def supervisor_node(state: StructuredState) -> dict:
””“Superviseur avec décision structurée.”””
decision: RouterDecision = router_llm.invoke([
HumanMessage(content=f”””
Tâche : {state[‘query’]}
Classification : {state[‘classification’]}
Quel agent appeler ?”””)
])
return {“next_agent”: decision.next_agent}

# ── Exemple direct ────────────────────────────────────────────
classification = classifier_llm.invoke([
HumanMessage(content=“Écris une fonction Python pour trier une liste”)
])

print(f”Type      : {type(classification).**name**}”)    # TaskClassification
print(f”Catégorie : {classification.category}”)          # code
print(f”Confiance : {classification.confidence:.0%}”)    # 95%
print(f”Sous-tâches : {classification.subtasks}”)

Pattern Réflexion (Reflection)

L'agent critique lui-même sa réponse

Le pattern Reflection consiste à faire générer une réponse par un LLM, puis à demander à un second LLM (ou au même avec un prompt différent) de la critiquer, et d'itérer jusqu'à ce que la qualité soit suffisante.

python — reflection.py 
from pydantic import BaseModel, Field
```

from typing import TypedDict, Annotated, Literal
from langgraph.graph import StateGraph, START, END, add_messages
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage

llm = ChatOpenAI(model=“gpt-4o”, temperature=0.3)

class CritiqueResult(BaseModel):
score:      int  = Field(ge=1, le=10)
is_good:    bool
weaknesses: list[str]
suggestions: list[str]

critique_llm = llm.with_structured_output(CritiqueResult)

class ReflectionState(TypedDict):
messages:    Annotated[list, add_messages]
task:        str
draft:       str
critique:    dict
iteration:   int
final:       str

# ── Générateur ────────────────────────────────────────────────
def generate(state: ReflectionState) -> dict:
context = ””
if state.get(“critique”):
c = state[“critique”]
context = f”””
Précédent brouillon : {state[‘draft’]}
Faiblesses identifiées : {c.get(‘weaknesses’, [])}
Suggestions : {c.get(‘suggestions’, [])}
Améliore le brouillon en corrigeant ces points.”””

```
resp = llm.invoke([
    SystemMessage(content="Expert rédacteur. Produis du contenu de haute qualité."),
    HumanMessage(content=f"Tâche : {state['task']}\n{context}")
])
return {
    "draft":     resp.content,
    "iteration": state["iteration"] + 1
}
```

# ── Critique ──────────────────────────────────────────────────
def critique(state: ReflectionState) -> dict:
result: CritiqueResult = critique_llm.invoke([
SystemMessage(content=””“Critique expert et exigeant.
Évalue ce texte sur 10. Sois dur mais constructif.”””),
HumanMessage(content=f”Tâche : {state[‘task’]}\n\nTexte :\n{state[‘draft’]}”)
])
print(f”  ↳ Score : {result.score}/10 | Bon : {result.is_good}”)
return {“critique”: result.model_dump()}

# ── Routeur ───────────────────────────────────────────────────
def should_revise(state: ReflectionState) -> Literal[“generate”, “finalize”]:
c = state.get(“critique”, {})
if state[“iteration”] >= 3:          # max 3 itérations
return “finalize”
if c.get(“is_good”, False):            # score suffisant
return “finalize”
return “generate”                       # encore améliorer

def finalize(state: ReflectionState) -> dict:
return {“final”: state[“draft”]}

# ── Assemblage ────────────────────────────────────────────────
builder = StateGraph(ReflectionState)
builder.add_node(“generate”, generate)
builder.add_node(“critique”, critique)
builder.add_node(“finalize”, finalize)

builder.add_edge(START, “generate”)
builder.add_edge(“generate”, “critique”)
builder.add_conditional_edges(“critique”, should_revise)
builder.add_edge(“finalize”, END)

reflection_app = builder.compile()

print(“🔄 Réflexion itérative…”)
result = reflection_app.invoke({
“messages”:  [],
“task”:      “Rédige une intro percutante sur LangGraph pour un blog technique”,
“draft”:     ””,
“critique”:  {},
“iteration”: 0,
“final”:     ””
})
print(f”\n✅ Résultat final ({result[‘iteration’]} itérations) :\n”)
print(result[“final”])
