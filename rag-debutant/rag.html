Comprendre les Embeddings — Le cœur du RAG

Pourquoi le texte devient des nombres, et comment ça marche
⏱ 35 min
▾
Théorie 15 min Pratique 20 min

📖 Concepts clés (15 min)

  • Qu'est-ce qu'un embedding ? (vecteur de 384 à 1536 dimensions)
  • Similarité cosinus : comment comparer deux textes
  • Pourquoi c'est supérieur à la recherche par mots-clés
  • Modèles gratuits vs OpenAI : quand utiliser quoi

💻 Exercice (20 min)

  • Installer sentence-transformers (gratuit, tourne en local)
  • Générer des embeddings sur 8 phrases métier
  • Trouver les 2 phrases les plus proches d'une requête
  • Observer : phrases similaires → vecteurs proches
jour1_embeddings.py
# pip install sentence-transformers scikit-learn
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

# Modèle 100% gratuit, tourne en local
model = SentenceTransformer(‘all-MiniLM-L6-v2’)

# Remplace par tes propres phrases métier !
corpus = [
“La politique de sécurité informatique de l’entreprise”,
“Procédure de gestion des incidents réseau”,
“Manuel d’onboarding des nouveaux employés”,
“Budget prévisionnel du département IT”,
“Règles de conformité RGPD et protection des données”,
]

query = “Comment gérer un incident de sécurité ?”

# Encoder tout d’un coup (rapide)
all_texts = [query] + corpus
vectors = model.encode(all_texts)

# Comparer la query (index 0) avec chaque doc
scores = cosine_similarity([vectors[0]], vectors[1:])[0]

print(”\n🔍 Résultats par pertinence :\n”)
for i in sorted(range(len(corpus)), key=lambda x: scores[x], reverse=True):
print(f”  {scores[i]:.3f}  →  {corpus[i]}”)
🎓
À enseigner

Analogie simple : l’embedding, c’est comme placer des mots dans un espace 3D où “chien” et “chat” sont proches, et “voiture” est loin. Montre la visualisation avec un scatter plot 2D (t-SNE).

Conseil Pro

Toujours ajouter des métadonnées dès le départ (source, date_création, auteur, département). Ça te permettra plus tard de filtrer : "cherche uniquement dans les docs RH de 2024".

jour2_chromadb.py
# pip install chromadb sentence-transformers
import chromadb
from sentence_transformers import SentenceTransformer

# Base persistante sur disque (réutilisable entre sessions)
client = chromadb.PersistentClient(path=”./base_rag”)
collection = client.get_or_create_collection(“docs_entreprise”)
model = SentenceTransformer(‘all-MiniLM-L6-v2’)

# Tes documents avec métadonnées riches
docs = [
{“text”: “Politique de télétravail : 2 jours max par semaine…”, “source”: “RH”, “date”: “2024-01”},
{“text”: “Procédure d’escalade incidents P1 : contacter le NOC…”, “source”: “IT”, “date”: “2024-03”},
{“text”: “Guide onboarding : accès VPN, badge, outils…”,     “source”: “RH”, “date”: “2024-02”},
]

# Insérer (avec check pour éviter les doublons)
existing = collection.get()[“ids”]
for i, doc in enumerate(docs):
doc_id = f”doc_{i}”
if doc_id not in existing:
collection.add(
documents=[doc[“text”]],
embeddings=model.encode([doc[“text”]]).tolist(),
metadatas=[{“source”: doc[“source”], “date”: doc[“date”]}],
ids=[doc_id]
)

# Requête avec filtre sur les métadonnées
results = collection.query(
query_embeddings=model.encode([“règles du télétravail”]).tolist(),
n_results=2,
where={“source”: “RH”}  # ← filtre par département !
)
print(results[“documents”], results[“metadatas”])

À enseigner

Compare ChromaDB à une bibliothèque avec un catalogue intelligent : au lieu de chercher par titre exact, tu décris ce que tu veux et le catalogue te trouve les livres les plus proches thématiquement.

Chunking Malin — La Qualité de Tes Réponses Vient de Là

Chunk size, overlap, stratégies avancées et métadonnées
⏱ 40 min
▾
Optimisation clé Pratique

📖 Stratégies de chunking (15 min)

  • Chunk trop petit → contexte insuffisant → réponse vague
  • Chunk trop grand → bruit → mauvaise précision
  • L'overlap : éviter de couper une idée en deux
  • MarkdownSplitter, SemanticChunker, RecursiveCharacter
  • Règle empirique : 300–600 tokens pour docs techniques

💻 Exercice comparatif (25 min)

  • Tester 3 tailles : 200 / 500 / 1000 tokens sur le même PDF
  • Poser la même question sur les 3 versions
  • Note tes observations dans un tableau
  • Implémenter le ParentDocument Retriever
🎓
À enseigner

Métaphore du cours magistral : un chunk = une diapo. Trop de texte par diapo → l'étudiant ne retient rien. Trop peu → il manque le contexte. Le bon chunk, c'est le bon résumé de diapo.

▾
Prompt Engineering Mémoire conversationnelle

📖 Concepts (15 min)

  • Le prompt template RAG : system + contexte + question
  • Instructions cruciales : "réponds uniquement avec le contexte"
  • ConversationalRetrievalChain : garder l'historique
  • ConversationBufferMemory vs WindowMemory
  • Gérer les questions de suivi ("Et pour ça ?", "Explique le point 2")

💻 Exercice (20 min)

  • Ajouter la mémoire conversationnelle

🎓
À enseigner

Le prompt template est l'instruction donnée au LLM. C'est la différence entre un assistant qui hallucine et un qui reste strict sur tes documents. Montrer côte à côte : sans instruction → invente. Avec → cite.
