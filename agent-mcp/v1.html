Agent vs Chatbot — La différence fondamentale

Boucle ReAct, raisonnement autonome, cycle Thought → Action → Observation
⏱ 45 min
▾
Théorie 20 min Code 25 min
```
💬

Chatbot (RAG)

Question → cherche → répond. Réactif. Une seule passe.

🤖

Agent

Objectif → raisonne → agit → observe → recommence. Autonome. Multi-passes.

🔁

Boucle ReAct

Reasoning + Acting. Le LLM alterne pensée et action jusqu'au but.

🛠️

Outil (Tool)

Fonction que l'agent peut appeler : calcul, recherche, API, base de données...

``` 🧠 THOUGHT : “Je dois trouver les règles VPN de l’entreprise, puis vérifier si elles s’appliquent au télétravail.” ↓ ⚡ ACTION : search_docs(“règles VPN entreprise”) ↓ 👁️ OBSERVATION : “VPN obligatoire hors bureaux, MFA requis, 2FA sur appareil pro uniquement” ↓ 🧠 THOUGHT : “Je dois maintenant chercher les spécificités télétravail.” ↓ ⚡ ACTION : search_docs(“politique télétravail accès distant”) ↓ 👁️ OBSERVATION : “Télétravail : VPN obligatoire + MFA, max 2j/semaine” ↓ ✅ FINAL ANSWER : “Oui, le VPN avec MFA s’applique aussi en télétravail…”
```
✅ Concepts vitaux
  • ReAct = Reason + Act en boucle, pas en ligne droite
  • L'agent décide lui-même quand s'arrêter (condition de fin)
  • max_iterations : toujours définir un plafond — sans lui l'agent tourne à l'infini
  • Un outil = une fonction Python décorée avec @tool
❌ Ignorer J1
  • Différences MRKL vs ReAct vs Reflexion (théorie pure)
  • Plan-and-Execute agents
  • OpenAI Function Calling vs Tool Calling
  • Prompt engineering avancé pour ReAct

# pip install langchain langchain-ollama langchain-core
# Prérequis : ollama serve  |  ollama pull mistral
# LangChain v1 : create_agent() remplace create_react_agent + AgentExecutor
from langchain.agents import create_agent   # ← nouvelle API v1 unifiée
from langchain_core.tools import tool
from langchain_ollama import ChatOllama

# ── DÉFINIR LES OUTILS : fonctions Python décorées @tool ──────────────
@tool
def calculatrice(expression: str) -> str:
    """Calcule une expression mathématique Python. Ex: '2 + 2', '15 * 3.5'"""
    try: return str(eval(expression, {"__builtins__": {}}, {}))
    except Exception as e: return f"Erreur : {e}"

@tool
def convertir_devises(montant: float, devise_source: str, devise_cible: str) -> str:
    """Convertit un montant entre EUR, USD, GBP."""
    taux = {("EUR","USD"):1.08, ("USD","EUR"):0.93, ("EUR","GBP"):0.86, ("GBP","EUR"):1.16}
    key = (devise_source.upper(), devise_cible.upper())
    if key in taux: return f"{montant * taux[key]:.2f} {devise_cible}"
    return f"Taux non disponible pour {devise_source}→{devise_cible}"

# ── CRÉER L'AGENT : create_agent() v1 ─────────────────────────────────
# Plus de prompt template manuel, plus d'AgentExecutor, plus de handle_parsing_errors
agent = create_agent(
    model=ChatOllama(model="mistral", temperature=0),
    tools=[calculatrice, convertir_devises],
    system_prompt="""Tu es un assistant de calcul et de change.
Utilise les outils disponibles pour répondre précisément.
Montre ton raisonnement avant chaque appel d'outil.""",
    debug=True,   # ← log chaque étape : nœuds model/tools dans la console
)

# ── INVOQUER : format standard messages list ───────────────────────────
taches = [
    "Si j'ai un budget de 1500 EUR, combien ça fait en USD ? Puis calcule 15% de cette somme.",
    "Calcule (256 * 4) + (1024 / 8) et dis-moi si c'est supérieur à 1100.",
]

for tache in taches:
    print(f"\n{'='*60}\n🎯 {tache}\n{'='*60}")
    result = agent.invoke({"messages": [{"role": "user", "content": tache}]})
    print(f"\n✅ RÉPONSE FINALE : {result['messages'][-1].content}")


# Avec debug=True tu vois dans la console :
# [model] tool_calls: [convertir_devises(montant=1500, devise_source="EUR", devise_cible="USD")]
# [tools] "1620.00 USD"
# [model] tool_calls: [calculatrice(expression="1620 * 0.15")]
# [tools] "243.0"
# [model] Final: "1500 EUR = 1620 USD, dont 15% = 243 USD"
```

📁 agents/jour2_memoire.py — Agent avec mémoire persistante JSON
``` 
# LangChain v1 : mémoire long terme via InMemoryStore (LangGraph store)
# pip install langchain langchain-ollama langgraph
import json
from dataclasses import dataclass
from langchain.agents import create_agent
from langchain.tools import tool, ToolRuntime   # ← ToolRuntime = accès au store depuis l'outil
from langchain_ollama import ChatOllama
from langgraph.store.memory import InMemoryStore  # ← store officiel LangGraph v1
from langgraph.checkpoint.memory import InMemorySaver

# ── CONTEXTE : identifie l'utilisateur courant ─────────────────────────
@dataclass
class Context:
    user_id: str   # passé à chaque invocation pour isoler les données par user

# ── STORE : base de mémoire long terme ────────────────────────────────
store = InMemoryStore()   # En prod → PostgresStore ou RedisStore

# ── OUTILS DE MÉMOIRE avec ToolRuntime ────────────────────────────────
# ToolRuntime injecte automatiquement le store et le contexte dans l'outil
@tool
def memoriser(fait: str, runtime: ToolRuntime[Context]) -> str:
    """Enregistre un fait important sur l'utilisateur pour s'en souvenir plus tard."""
    # Namespace = (user_id, "faits") — chaque user a son espace isolé
    existing = runtime.store.get(("faits",), runtime.context.user_id)
    faits = existing.value["liste"] if existing else []
    faits.append(fait)
    runtime.store.put(("faits",), runtime.context.user_id, {"liste": faits})
    return f"✓ Mémorisé : {fait}"

@tool
def rappeler(sujet: str, runtime: ToolRuntime[Context]) -> str:
    """Cherche dans la mémoire persistante des infos sur un sujet."""
    existing = runtime.store.get(("faits",), runtime.context.user_id)
    if not existing: return "Aucun souvenir encore."
    faits = [f for f in existing.value["liste"] if sujet.lower() in f.lower()]
    return "\n".join(faits) if faits else f"Aucun souvenir sur '{sujet}'"

# ── CRÉER L'AGENT avec store ────────────────────────────────────────────
agent = create_agent(
    model=ChatOllama(model="mistral", temperature=0),
    tools=[memoriser, rappeler],
    system_prompt="Tu es un assistant avec mémoire persistante. Utilise memoriser() pour tout fait important. Utilise rappeler() pour retrouver une info.",
    store=store,                   # ← store partagé avec les outils via ToolRuntime
    checkpointer=InInMemorySaver(),  # ← nécessaire pour la persistance de session
    context_schema=Context,        # ← schéma du contexte injecté dans ToolRuntime
)

# ── SESSION 1 : l'agent apprend ────────────────────────────────────────
config = {"configurable": {"thread_id": "s1", "context": Context(user_id="mohamed")}}
agent.invoke({"messages": [{"role": "user", "content": "Je m'appelle Mohamed, je travaille chez JEMS Group en DevOps. Mémorise ça."}]}, config)

# ── SESSION 2 : l'agent se souvient (nouveau thread_id, même user_id) ─
config2 = {"configurable": {"thread_id": "s2", "context": Context(user_id="mohamed")}}
result = agent.invoke({"messages": [{"role": "user", "content": "Où est-ce que je travaille ?"}]}, config2)
print(f"\n✅ {result['messages'][-1].content}")


# Sortie attendue :
# Session 1 : "✓ Mémorisé : Mohamed travaille chez JEMS Group en DevOps"
# Session 2 : "Tu travailles chez JEMS Group en tant que DevOps."
# La mémoire persiste grâce au store InMemoryStore — chaque user_id est isolé.
```
🎓 Pour enseigner

Lance les deux sessions l'une après l'autre en live. Le "wow" : session 2 répond correctement sans que l'info ait été donnée dans ce thread. Montre que thread_id="s2" est différent — c'est bien le store (et non l'historique de conversation) qui persiste la mémoire. Clé pédagogique : store = mémoire inter-sessions, checkpointer = mémoire intra-session.

✅ Checklist — Passer au J3

  • Session 2 (thread_id différent) retrouve les faits de session 1
  • Tu comprends la différence store (inter-sessions) vs checkpointer (intra-session)
  • Tu peux expliquer les 4 types de mémoire et quand les utiliser

from pathlib import Path
from pydantic import BaseModel, Field
from langchain_core.tools import tool, StructuredTool
from langchain_ollama import ChatOllama

# ── OUTIL 1 : API HTTP avec timeout et gestion d’erreur ───────────────
@tool
def meteo_ville(ville: str) -> str:
””“Retourne la météo actuelle d’une ville française (Open-Meteo, gratuit, sans clé).”””
# Geocoding d’abord
geo = requests.get(
f”https://geocoding-api.open-meteo.com/v1/search?name={ville}&count=1&language=fr”,
timeout=10
).json()
if not geo.get(“results”): return f”Ville ‘{ville}’ non trouvée.”
lat, lon = geo[“results”][0][“latitude”], geo[“results”][0][“longitude”]
# Météo
data = requests.get(
f”https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}¤t=temperature_2m,wind_speed_10m”,
timeout=10
).json()[“current”]
return f”{ville} : {data[‘temperature_2m’]}°C, vent {data[‘wind_speed_10m’]} km/h”

# ── OUTIL 2 : Lecture CSV avec validation Pydantic ─────────────────────
class LectureCSVInput(BaseModel):
chemin: str = Field(description=“Chemin relatif vers le fichier CSV”)
max_lignes: int = Field(default=10, description=“Nombre max de lignes à retourner”, ge=1, le=100)

def lire_csv_fn(chemin: str, max_lignes: int = 10) -> str:
# Sécurité : restreindre aux fichiers dans le répertoire courant
p = Path(chemin).resolve()
if not str(p).startswith(str(Path(”.”).resolve())):
return “❌ Accès refusé : chemin hors du répertoire autorisé.”
if not p.exists(): return f”Fichier introuvable : {chemin}”
rows = list(csv.DictReader(p.open(encoding=“utf-8”)))
sample = rows[:max_lignes]
return json.dumps(sample, ensure_ascii=False, indent=2)

lire_csv = StructuredTool.from_function(
func=lire_csv_fn,
name=“lire_csv”,
description=“Lit et retourne le contenu d’un fichier CSV. Utile pour analyser des données tabulaires.”,
args_schema=LectureCSVInput
)

# ── OUTIL 3 : Sauvegarder un rapport ──────────────────────────────────
@tool
def sauvegarder_rapport(nom_fichier: str, contenu: str) -> str:
””“Sauvegarde un rapport ou résumé dans un fichier Markdown.”””
if not nom_fichier.endswith(”.md”): nom_fichier += ”.md”
Path(nom_fichier).write_text(contenu, encoding=“utf-8”)
return f”✓ Rapport sauvegardé : {nom_fichier}”

# Test rapide des outils sans agent
print(meteo_ville.invoke(“Paris”))
print(lire_csv.invoke({“chemin”: “employes.csv”, “max_lignes”: 5}))

✅ Pourquoi LangGraph ?
  • create_agent() = rapide et suffisant pour 80% des cas. LangGraph = contrôle total pour les workflows complexes
  • État (State) : une TypedDict qui circule entre les nœuds
  • Nœuds (Nodes) : fonctions Python qui modifient l'état
  • Arêtes (Edges) : conditions de routage entre nœuds
  • END : nœud terminal, la sortie du graphe
❌ Ignorer J5
  • Subgraphs et graphes imbriqués
  • Checkpointing avec SQLite/PostgreSQL
  • Streaming token par token
  • LangGraph Studio (UI de debug)
``` TypedDict State : { messages: list, outils_utilisés: list, réponse_finale: str } ┌─────────────┐ │ START │ └──────┬──────┘ ↓ ┌─────────────┐ │ llm_node │ ← LLM décide : répondre ou appeler un outil └──────┬──────┘ ↓ condition ┌────┴────┐ ↓ ↓ tool_node END ← outil appelé → résultat injecté dans State → retour au LLM ↓ └────→ llm_node (boucle)
```
📁 agents/jour5_langgraph.py — Premier graphe LangGraph avec Mistral
# pip install langgraph langchain-ollama langchain-core
```

from typing import TypedDict, Annotated, Sequence
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langchain_core.tools import tool
from langchain_ollama import ChatOllama
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
import operator

# ── 1. DÉFINIR L’ÉTAT ─────────────────────────────────────────────────
# L’état circule entre tous les nœuds — c’est la mémoire du graphe
class AgentState(TypedDict):
# Annotated + operator.add = les messages s’accumulent (pas de remplacement)
messages: Annotated[Sequence[BaseMessage], operator.add]

# ── 2. OUTILS ─────────────────────────────────────────────────────────
@tool
def calculatrice(expression: str) -> str:
””“Calcule une expression mathématique.”””
try: return str(eval(expression, {”**builtins**”: {}}, {}))
except Exception as e: return f”Erreur : {e}”

tools = [calculatrice]
llm = ChatOllama(model=“mistral”, temperature=0).bind_tools(tools)  # ← bind_tools v1

# ── 3. NŒUDS : fonctions qui modifient l’état ─────────────────────────
def llm_node(state: AgentState) -> dict:
“Le LLM reçoit tous les messages et génère une réponse (ou un appel d’outil)”
response = llm.invoke(state[“messages”])
return {“messages”: [response]}  # ajouté à la liste via operator.add

tool_node = ToolNode(tools)  # nœud prêt-à-l’emploi qui exécute les outils

# ── 4. CONDITION DE ROUTAGE ────────────────────────────────────────────
def doit_continuer(state: AgentState) -> str:
“Si le dernier message contient un appel d’outil → tools. Sinon → END”
last = state[“messages”][-1]
if hasattr(last, “tool_calls”) and last.tool_calls:
return “tools”
return END

# ── 5. CONSTRUIRE LE GRAPHE ────────────────────────────────────────────
graph = StateGraph(AgentState)
graph.add_node(“llm”, llm_node)
graph.add_node(“tools”, tool_node)
graph.set_entry_point(“llm”)
graph.add_conditional_edges(“llm”, doit_continuer, {“tools”: “tools”, END: END})
graph.add_edge(“tools”, “llm”)  # après outil → retour au LLM
app = graph.compile()

# ── 6. INVOQUER ───────────────────────────────────────────────────────

from langchain_core.messages import BaseMessage, HumanMessage
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_ollama import ChatOllama
from langchain_chroma import Chroma
from langchain_huggingface import HuggingFaceEmbeddings
from langgraph.graph import StateGraph, END
import operator

llm = ChatOllama(model=“mistral”, temperature=0)
emb = HuggingFaceEmbeddings(model_name=“all-MiniLM-L6-v2”)
vectordb = Chroma(persist_directory=”./base_entreprise”, embedding_function=emb)

# ── ÉTAT ÉTENDU ───────────────────────────────────────────────────────
class State(TypedDict):
messages: Annotated[Sequence[BaseMessage], operator.add]
categorie: str          # “it”, “rh”, “calcul”, “general”
contexte_docs: str      # chunks RAG récupérés
reponse: str            # réponse finale

# ── NŒUD 1 : CLASSIFIER ───────────────────────────────────────────────
def classifier_node(state: State) -> dict:
question = state[“messages”][-1].content
prompt = ChatPromptTemplate.from_messages([
(“system”, “Classe la question en UNE seule catégorie : it | rh | calcul | general. Réponds uniquement avec la catégorie.”),
(“human”, ”{q}”),
])
cat = (prompt | llm | StrOutputParser()).invoke({“q”: question}).strip().lower()
return {“categorie”: cat if cat in [“it”,“rh”,“calcul”] else “general”}

# ── NŒUD 2 : RAG (IT + RH combinés ici pour simplifier) ───────────────
def rag_node(state: State) -> dict:
question = state[“messages”][-1].content
docs = vectordb.as_retriever(search_kwargs={“k”:3}).invoke(question)
ctx = ”\n\n”.join(f”[{d.metadata.get(‘source’,’?’)}] {d.page_content}” for d in docs)
return {“contexte_docs”: ctx}

# ── NŒUD 3 : SYNTHÈSE FINALE ──────────────────────────────────────────
def synthese_node(state: State) -> dict:
question = state[“messages”][-1].content
prompt = ChatPromptTemplate.from_messages([
(“system”, “Réponds à la question avec le contexte fourni. Si pas de contexte, réponds avec tes connaissances.\n\nContexte:\n{ctx}”),
(“human”, ”{q}”),
])
reponse = (prompt | llm | StrOutputParser()).invoke({“ctx”: state.get(“contexte_docs”,””), “q”: question})
return {“reponse”: reponse}

# ── ROUTAGE : selon la catégorie ──────────────────────────────────────
def router(state: State) -> str:
cat = state.get(“categorie”, “general”)
return “rag” if cat in [“it”, “rh”] else “synthese”  # calcul et general → direct synthese

# ── CONSTRUIRE LE GRAPHE ──────────────────────────────────────────────
g = StateGraph(State)
g.add_node(“classifier”, classifier_node)
g.add_node(“rag”, rag_node)
g.add_node(“synthese”, synthese_node)
g.set_entry_point(“classifier”)
g.add_conditional_edges(“classifier”, router, {“rag”: “rag”, “synthese”: “synthese”})
g.add_edge(“rag”, “synthese”)
g.add_edge(“synthese”, END)
app = g.compile()

# ── TEST ──────────────────────────────────────────────────────────────
for q in [“Quelles sont les règles VPN ?”, “Calcule 15% de 2400”, “Qu’est-ce que le RGPD ?”]:
r = app.invoke({“messages”: [HumanMessage(content=q)], “categorie”: ””, “contexte_docs”: ””, “reponse”: ””})
print(f”\n[{r[‘categorie’].upper()}] {q}\n→ {r[‘reponse’][:200]}”)
